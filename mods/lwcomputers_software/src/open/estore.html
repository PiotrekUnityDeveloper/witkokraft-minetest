estore

The eStore application receives orders from the eShop application running on a client. On receipt a bank transfer for payment is made and the order is accepted. Both the eShop client's bank account and the eStore retailer's bank account must be with the same bank. The order then must be filled and the order on the register dismissed with the New button. Any unsupplied items must be manualy refunded.

Requires WIN, register.

A settings file of the same name in the same folder, appended with .ini, containing the following settings.

register The path to the register application,
         defaults to /apps/register.

The eStore application accesses the register application's .ini and .dat files.
